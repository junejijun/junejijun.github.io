面向对象基础篇——Java（三）

1.对象与类

1.1对象

  • 对象是一种数据结构，本质上就是一种数据的组织方式；
  • 在内存上，对象就是一个内存块，存放相关的数据集合；

1.2类

1.2.1定义

类是用于描述同一类型的对象的一个抽象概念，类中定义了这一类对象所对应的共同属性和方法；

1.2.2 组成

对于一个类来说，一般有三种常见的成员：属性field、方法method和构造器constructor；

1.2.3 实例

1.3两者之间的关系

  • 对象是具体的事物，类是对对象的抽象；
  • 类可以看成一类对象的模板，对象可以看做该类的一个具体实例；

2.面向对象的内存分析

Java虚拟机的内存可以分为三个区域：栈stack、堆heap和方法区method area；

2.1栈的特点

  • 方法执行的内存模型，每个方法被调用在栈中都会创建一个栈帧；
  • JVM为每个线程都创建一个栈，栈属于线程私有，不同线程无法实现栈的共享；
  • 栈的存储特点是“先进后出，后进先出”；
  • 栈是一个连续的内存空间，由系统自动分配；

2.2堆的特点

  • 堆用于存储创建好的对象；
  • JVM只有一个堆，被所有线程共享
  • 堆是一个不连续的内存空间；

2.3方法区（静态区）的特点

  • 方法区实际上属于堆，用于存储程序中永远不变或唯一的内容，如静态变量、静态方法和字符串常量等；
  • JVM只有一个方法区，被所有线程共享；

3.this关键字

3.1 对象创建的过程

创建一个对象分为如下四步：

  1. 分配对象空间，并将对象成员变量进行初始化；
  2. 执行属性值的显示初始化；
  3. 执行构造方法；
  4. 返回对象的地址给相关的变量；

3.2this的本质

this的本质就是“创建好的对象的地址”，由于在构造方法调用前，对象就已经创建，因此可以在构造方法中使用this代表“当前对象”；

3.3 this的用法

3.3.1this代表“当前对象”

在程序中产生歧义时，应使用this来指明；

在普通方法中 ，this总是指向调用该方法的对象；

在构造方法中，this总是指向正要初始化的对象；

4.static关键字

4.1static使用特点

  • 用static声明的成员变量为静态成员变量，也称类变量；
  • 静态成员变量被该类的所有对象共享；
  • 可以直接通过类名来访问静态成员；
  • 在static方法中不可以直接访问非static的成员；

4.2static核心要点

  • static修饰的成员变量和方法，从属于类；
  • 普通变量和方法从属于对象；
