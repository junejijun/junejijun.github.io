Postman使用教程

2022-06-05

1. Postman工具介绍

1.1 Workspaces界面

  • ==Collection==：接口测试项目

  • APIs：API文档

  • ==Environments==：环境变量、全局变量

  • ==Mock Servers==：模拟服务器

  • Monitors：监听器

  • History：请求的历史记录

1.2 请求页签界面

  • ==Params：get请求传参参数==

  • Authorization：鉴权，常用Type是Bearer Token和Basic Auth

  • Headers：输入请求头

  • ==Body：post请求传参参数==

  • Pre-request Script：请求之前需要执行的脚本

  • Tests：请求之后的断言

  • Settings：请求相关设置

  • none：没有参数
  • ==form-data==：既可以传键值对参数，也可以传文件
  • ==x-www-form-urlencoded==：只能传键值对参数
  • ==raw==：可以传多种类型的参数，如JSON、JavaScript、Text、HTML、XML
  • ==binary==：把文件以二进制的形式进行传参
  • GraphQL（用的比较少）

1.3 响应页签界面

  • ==Body：接口返回的数据==
    • Pretty：支持以不同的格式查看返回的数据，如JSON、Text、HTML、XML
    • Raw：以文本的形式查看返回的数据
    • Preview：以网页的形式查看返回的数据
  • Cookies：响应的Cookies信息
  • Headers：响应头
  • ==Test Results：断言的结果==

2. 常用功能演示

2.1 环境变量与全局变量

2.1.1 环境变量

解决的问题：针对不同环境（如开发环境、测试环境、生产环境等），其请求的IP地址是不同的。

解决的方案：

  • 第一步：设置环境变量，VARIABLE名称要一致
  • 第二步：以的形式替换url中的IP地址来获取对应环境下的环境变量值
  • 第三步：选择对应的环境变量

2.1.2 全局变量

解决的问题：（1）接口A中响应的参数需作为接口B请求的参数输入，详细解决的方案见【接口关联】；（2）自定义动态参数设置，详细解决的方案见【自定义动态参数】。

2.2 接口关联

解决的问题：接口A中响应的参数需作为接口B请求的参数输入。

解决的方案：

  • 第一步：在接口A请求页签Tests中编写断言，支持使用JSON提取器与正则表达式两种方式实现接口关联；

  • 第二步：在接口B请求页签Params中以的形式使用；

2.4 断言

==状态断言一般只有一个，且可以设置为全局断言==

==业务断言可以有多个==

  • Status Code：Code is 200 断言返回的状态码为200
  • Response body：Contains string 断言返回的body中包含有一个字符串
  • Response body：JSON value check 检查json中的其中的一个值
  • Response body：is equal a string 断言返回的body等于一个字符串
  • Response headers：断言响应头包含Content-Type
  • Response time is less than 200ms 断言请求的时间少于200ms

批量运行接口时，文件上传接口报错问题解决方案：

  • 第一步：将所需上传的文件复制到C:\Users\June\Postman\files

  • 第二步：设置允许读取外部目录文件

2.5 CSV或JSON文件实现数据驱动

  • 第一步：创建CSV或JSON文件

  • 第二步：修改请求参数的值，全部以的形式赋值

  • 第三步：修改请求页签中Tests的内容

  • 第四步：点击Select File选择CSV或JSON文件驱动数据

2.6 Mock Servers

使用场景：前后端分离，后端的接口没有完成，但是前端已经完成了，且前端的业务流程依赖于后端的接口。

创建一个新的mock server后，软件会自动创建对应的环境变量与Collection
